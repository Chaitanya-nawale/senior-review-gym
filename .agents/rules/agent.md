# agent.md — Developer Guidelines & Project Reference

## Project Overview

**MeisterUp** is an AI-native adaptive learning platform for engineers. The current codebase is the **Senior Review Gym** — an interactive Code Comprehension Gym built as a swiping-deck UI for code review practice.

**MeisterUp** models your existing knowledge, finds the exact gaps that matter, and generates a personalized curriculum for any technical skill — from Rust to system design to prompt engineering.

## Key Architectural Notes

### Vite Config — Do NOT add these plugins manually

`@lovable.dev/vite-tanstack-config` already bundles:

- TanStack devtools, `tanstackStart`, `viteReact`, `tailwindcss`, `tsConfigPaths`
- Nitro (build-only, Cloudflare target), VITE\_\* env injection, `@` path alias
- React/TanStack dedupe, error logger, sandbox detection

Adding any of these again **will break the build**.

### Routing (File-Based)

- `src/routes/__root.tsx` — root shell with global providers, fonts (Inter + JetBrains Mono), and error/404 boundaries
- `src/routes/index.tsx` — the entire application UI (landing page + gym component)
- New routes → create new files under `src/routes/`; the route tree is auto-generated into `src/routeTree.gen.ts` (do not edit manually)

### Styling

- Tailwind CSS v4 with a dark theme (html has `class="dark"`)
- Base body: `bg-black text-gray-100`
- Google Fonts loaded in `__root.tsx`: **Inter** (UI) + **JetBrains Mono** (code)
- Design tokens live in `src/styles.css`

### SSR

- Server entry: `src/server.ts` (wraps Nitro/Cloudflare with error reporting)
- Client hydration via `src/start.ts`

## Design Conventions

- **Dark mode only** — no light mode toggle
- Animations via Framer Motion (`motion.*`, `AnimatePresence`)
- Swipe interactions use `useMotionValue` + `useTransform` for drag gestures
- Icons from `lucide-react` only
- Component utilities: `cn()` helper (clsx + tailwind-merge) defined locally in `src/routes/index.tsx` and also available via `src/lib/utils.ts`
